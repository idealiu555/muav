# 介绍包装

## 介绍

这个项目我把低空网络中的通信保障建模成多智能体的飞行轨迹与 3D 波束赋形联合优化问题，让多架 UAV 在每个时隙同时决策位移和波束方向，并在系统速率、服务时延、能耗和公平性之间做权衡。针对局部观测里邻居 UAV 和关联 UE 数量可变、固定张量输入必须截断填充的问题，我在 MAPPO 中加入了 attention encoder，用 count 生成 mask，再用 cross-attention 只聚合真实实体信息，缓解 padding 带来的信息损失；同时在 centralized critic 中引入 one-hot 身份标识，进一步细化价值估计粒度。实验上，相对 MAPPO，系统速率提升了 2.61%，服务时延降低了 10.05%，说明可以在一定程度上学到波束指向和多智能体协作。

## 包装

{AMAPPO: Attention Enhanced MAPPO for Joint Optimization of Trajectory and Beamforming in Low-Altitude Networks, IEEE Communications Letters, Pending, \textbf{一作}.}

将低空网络中的通信保障建模为\textbf{多智能体的飞行轨迹与3D波束赋形联合优化问题}，设计注意力机制增强的\textbf{多智能体强化学习算法}：利用Cross-Attention处理观测截断，引入身份标识优化价值分解。实验表明AMAPPO\textbf{对比MAPPO提升系统速率2.61\%，降低服务时延10.05\%}。\\[0.85ex]

# 面试高频问答

# AMAPPO 面试高频问题准备

## 问题1：你这个项目的核心研究问题是什么，为什么要做联合优化？

这个项目的核心问题是，在低空网络里，多架 UAV 既要通过移动去覆盖用户，又要通过 3D 波束赋形提高链路质量，这两个决策本质上是强耦合的。单独优化轨迹，可能把 UAV 移到更好的覆盖位置，但波束方向不合理时吞吐和干扰控制仍然不理想；只优化波束，又无法解决 UE 分布变化带来的覆盖与回传压力。所以我把它统一建模成多智能体联合决策问题，让每架 UAV 在每个时隙同时输出三维位移和波束方向，从系统速率、服务时延、能耗和公平性四个维度共同优化，这样更符合真实低空通信场景。

## 问题2：为什么你选择 MAPPO 作为主算法，而不是 MADDPG 或其他方法？

我最终选择 MAPPO，主要是因为这个场景里多智能体之间的耦合很强，环境动态也比较复杂，PPO 类方法在训练稳定性上通常更可控。相比 MADDPG 这种偏确定性策略的方法，MAPPO 对高维连续动作和多目标奖励的适应性更好，策略更新不会因为 Q 值估计误差被快速放大。另外我这里动作既包含轨迹控制，也包含波束角控制，策略分布需要有足够的探索弹性，MAPPO 的 stochastic policy 更适合做这种联合优化。在工程上，我也对 MAPPO 做了有界动作建模、价值归一化和梯度裁剪，所以它比纯 baseline 更稳定，适合做主方法。

## 问题3：你的状态空间和动作空间是怎么定义的？

每个智能体对应一架 UAV，局部观测主要由四部分构成：第一部分是自身状态，包括归一化三维位置、缓存状态和活跃标记；第二部分是邻居 UAV 信息，包括相对位置、缓存位图、互补性和可协作能力；第三部分是当前服务 UE 的信息，包括相对位置、请求文件 ID 和本地缓存命中标志；第四部分是邻居数量和关联 UE 数量这两个计数字段，用来给 attention 模块生成 mask。动作空间是 5 维连续动作，前三维控制 x、y、z 三个方向的归一化位移，后两维控制波束俯仰角和方位角。这样设计的目的是让每架 UAV 同时感知覆盖、协作和业务需求，并把移动与通信方向统一到同一个决策闭环里。

## 问题4：Attention 在这个项目里具体解决了什么问题？为什么用 Cross-Attention？

这个场景的一个关键难点是局部观测不是固定实体集合。不同 UAV 在不同时隙看到的邻居数量、服务 UE 数量都不一样，但神经网络输入又必须是固定维度，所以环境里只能先做截断和 padding。问题在于，如果限定观测数量较少，会导致实际服务UE数量大于观测到的UE数量，也就是观测截断的问题，这样会让智能体对服务UE所需的资源估计不准。而如果限定观测数量较多，会导致观测空间中有很多空位，如果直接把这些空填充送进普通 MLP，模型很容易把无效槽位也当成真实信息学习进去。我这里的做法是保留 count 字段生成 mask，然后用 UAV 自身状态作为 Query，邻居和 UE 特征作为 Key 和 Value 做 Cross-Attention，只聚合当前时刻真正有效的实体信息，并通过多层注意力迭代细化聚合结果。这样既能抑制 padding 噪声，又能让模型学会“当前这架 UAV 最该关注谁”，比简单拼接或均值池化更合理。

## 问题5：你提到引入 one-hot 身份标识来细化价值估计，这一步为什么有效？

MAPPO 的一个常见问题是 centralized critic 容易学成比较粗的团队价值，尤其在多 UAV 场景下，不同智能体虽然共享同一个全局状态，但承担的位置、邻域关系和风险并不一样。如果 critic 不知道“当前在评估谁”，就容易把多个 agent 的价值混在一起，导致优势函数不够细。我的做法是先基于所有 agent 的 attention 编码构建团队上下文 `team_context`，再在 critic 端拼接当前 agent 的 one-hot 身份标识，让 critic 在共享团队信息的同时保留个体索引，从而学习到更细粒度的 value decomposition。这样做的直观收益是，对边界风险、局部拥塞、协作压力不同的 UAV，critic 能给出更有区分度的价值估计，策略更新也会更准确。

## 问题6：你的奖励函数是怎么设计的？怎么避免模型只追求某一个指标？

我的奖励设计是一个多目标加权形式，核心思想不是只追求吞吐最大，而是在系统速率、服务时延、能耗和公平性之间做平衡。具体做法是把总时延和总能耗作为惩罚项，把系统速率和 Jain 公平性指数作为奖励项，再通过尺度归一化和对数压缩把不同量纲拉到接近的范围，避免某一个指标主导训练。另外我还把碰撞、危险接近和越界设计成显式惩罚，保证策略不会为了追求短期吞吐做出不安全动作。这样训练出来的策略不是单点最优，而是更符合通信系统实际需求的综合最优。

## 问题7：3D 波束赋形在你的环境里是怎么建模的？和轨迹优化怎么耦合？

在环境里，波束控制不是抽象标签，而是有明确物理语义的。每架 UAV 除了移动动作，还要输出两个连续控制量，对应俯仰角和方位角，再映射成实际的 3D 波束方向。链路增益计算时会考虑 UAV 到 UE 的相对几何关系、LoS 概率、路径损耗以及波束方向和目标方向之间的夹角，所以波束并不是独立模块，而是直接影响 UE-UAV 链路的信道质量和同频干扰。它和轨迹是强耦合的，因为 UAV 位置一旦变化，最优波束方向、干扰关系和覆盖集合都会同步变化。也正因为这种耦合，联合优化比先定轨迹再调波束更有意义。

## 问题8：你怎么证明性能提升确实来自 attention 和身份标识，而不是训练波动？

我验证贡献主要靠两类实验。第一类是和标准 MAPPO 做同条件对比，保证环境、随机种子、训练轮数、动作空间和奖励函数保持一致，只替换算法结构；这样可以直接看出 AMAPPO 相比 baseline 在系统速率和服务时延上的改善。第二类是消融实验，把 attention 和身份标识分别拿掉，比较只加 attention、只加 identity、两者都加三种配置的差异，观察性能变化和收敛曲线。我的判断标准不只看最终均值，还看训练过程是否更稳、波动是否更小。如果一个模块只在个别 seed 上有效，我不会把它作为核心结论，而是看它是否在整体趋势上稳定贡献收益。

## 问题9：为什么你的系统速率只提升了 2.61%，但时延下降了 10.05%？

我对这个结果的理解是，attention 和身份标识带来的收益更多体现在“资源调度更稳、更细”，而不是单纯把链路峰值速率拉高。低空网络里的时延不仅取决于瞬时吞吐，还和 UE 关联是否合理、协作链路是否顺畅、缓存命中与否、干扰是否被压制有关。AMAPPO 通过更准确地关注有效 UE 和邻居关系，能减少一些低效覆盖、错误协作和无意义移动，所以对服务时延的改善更明显。吞吐提升只有 2.61% 并不说明方法弱，反而说明这个场景已经比较饱和，想继续抬高峰值速率很难，而降低系统性时延更能体现算法的价值。

## 问题10：这个项目里你觉得最有技术含量、也最容易被追问的点是什么？

我觉得最有技术含量的不是单独把 attention 或 PPO 套进去，而是把“低空通信系统建模”和“多智能体学习机制”真正对齐。前者要求你把移动、波束、干扰、回传、缓存和安全约束放到同一个环境里，后者要求你考虑可变长度观测、集中训练分布执行以及多智能体价值分解。我这个工作的关键就在于，先把问题建模成一个合理的联合优化任务，再针对观测截断和价值粒度两个真正影响学习效果的瓶颈，分别用 Cross-Attention 和 one-hot identity 去处理。面试时如果展开讲，我会把贡献概括成一句话：不是简单换模型，而是针对场景结构去改算法。
